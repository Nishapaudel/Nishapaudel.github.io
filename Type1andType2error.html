Understanding Type I and Type II Errors (with Simple Example)

Published on: May 10, 2025

In statistics, we often want to know whether something is true — for example, whether there are fish in a pond. To do this, we use hypothesis testing. But our conclusions are based on samples, not certainty, which means we can make mistakes. These mistakes are known as Type I and Type II errors.

Setting Up the Hypotheses

Let's say you're testing whether there are fish in a pond. You set up your hypotheses as:

  • Null Hypothesis (H₀): There are no fish in the pond.
  • Alternative Hypothesis (H₁): There are fish in the pond.

You then fish for a while and base your conclusion on whether or not you catch something, and the probability of catching something if there were really no fish (this is the p-value).

Numerical Example: Making a Decision

Imagine you cast a net into the pond for 10 minutes:

  • Case A: You catch 1 fish. A statistical test gives a p-value = 0.01
  • Since 0.01 < α = 0.05 (5% significance level), you reject H₀. You conclude: "There are fish in the pond." ✅

Case B: You catch nothing. Your test gives a p-value = 0.20

  • Since 0.20 > 0.05, you fail to reject H₀. You conclude: "There’s not enough evidence that fish are in the pond." ❌ (but there might still be fish!)

Visualizing with a Confusion Matrix

We Say: "Fish Exist"
(Reject H₀)		 ✅ True Positive: We caught a fish and we're correct ❌ Type I Error: We thought we caught a fish, but it was a crab (false positive)
We Say: "No Fish"
(Fail to Reject H₀)		 ❌ Type II Error: There were fish but we didn’t catch any (false negative) ✅ True Negative: We didn’t catch anything because the pond really has no fish

Type I Error (False Positive)

Occurs when we reject the null hypothesis even though it’s actually true. In our example, that’s saying “There are fish” when there aren’t. You caught a crab and mistook it for a fish.

Type I Error = Catching a crab, thinking it’s a fish.

Type II Error (False Negative)

Occurs when we fail to reject the null hypothesis even though the alternative is true. That means there were fish, but you didn’t catch any and concluded the pond is empty.

Type II Error = There are fish, but we missed them.

How to Reduce These Errors

  • Lower Type I Error: Use a smaller significance level (e.g., 0.01 instead of 0.05). Be more strict before saying fish exist.
  • Lower Type II Error: Increase fishing effort (larger sample size). The more time you spend fishing, the more likely you’ll detect a real fish.

Confidence Intervals

Another way to decide is using confidence intervals (CIs). If the 95% CI for the number of fish excludes 0, we reject H₀. If it includes 0, we cannot say fish exist.

  • CI = (2, 5) → no 0 in range → Reject H₀
  • CI = (-1, 3) → 0 is inside → Fail to reject H₀

Conclusion

Type I and Type II errors help us understand the risks of wrong conclusions in hypothesis testing. Using examples like fishing makes it easier to visualize the logic. Always consider both types of error and use good design (like adequate sample size) for reliable decisions.
